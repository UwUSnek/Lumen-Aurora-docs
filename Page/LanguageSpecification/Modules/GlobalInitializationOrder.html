The global initialization order defines the order in which variables declared outside of routines are initialized. This only affects global variables with an initializer value and enum elements.

Global variables and enum elements of a module are initialized in the same order in which they are declared.
The order in which modules are initialized depends on the import graph of the program: each module is guaranteed to be initialized before any module that depends on it.

The initialization order is only defined when there are no circular dependencies in the import graph.
Circular module dependencies cause all modules in the cycle to have no defined initialization order relative to each other, though they are still guaranteed to be initialized before modules that depend on them but are not in the cycle.

Circular dependencies between initializer values are forbidden.

Example 1
import io; // int a = b; Error, circular dependency of initializer values // int b = a; Error, circular dependency of initializer values uint a = 8; // uint b = main.e.n; Error, main.e.n is not initialized here void main() { io.println("The value of a is: " + a); io.println("The value of e.n is: " + e.n); io.println("The value of b is: " + b); io.println("main function"); enum e : int { n = -6; } } uint b = main.e.n; // Ok, main.e.n is initialized here
Output
The value of a is: 8 The value of e.n is: -6 The value of b is: -6 main function
Example 2 - test_0.lmn
import io; t log_and_return(t? value, str name) { io.println("initialized \"" + name + "\""); return value; } int a = log_and_return(0, "test_0.a");
Example 2 - test_a.lmn
import "test_0.lmn"; int a = test_0.log_and_return(0, "test_a.a");
Example 2 - test_b.lmn
import "test_0.lmn"; int a = test_0.log_and_return(0, "test_b.a");
Example 2 - main.lmn
import "test_b.lmn"; // test_b is imported before test_a, so it's initialized first import "test_a.lmn"; import "test_0.lmn"; // Initialized first as test_a and test_b depend on it import io; int b = test_0.log_and_return(12, "b"); void main() { io.println("main function"); }
Output
initialized "test_0.a" initialized "test_b.a" initialized "test_a.a" initialized "b" main function

Global variables are initialized by a special __init function which cannot be called directly. This is where execution actually begins. After all global variables and enum elements have been initialized, the main function is called.

const variables and enum elements are technically initialized during the compilation phase and often optimized out completely, albeit without changing the observable behaviour of the program.
This means that const values are always available to non-const expressions, but for the purposes of initialization, they behave as if initialized during the global initialization stage. This ensures the initialization rules are kept simple and consistent.
