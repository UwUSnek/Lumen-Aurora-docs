The global initialization order defines the order in which variables declared outside of routines are initialized. This only affects global variables with an initializer value.

Global variables of a module are initialized in the same order in which they are declared.
The order in which modules are initialized depends on the import graph of the program: each module is guaranteed to be initialized before any module that depends on it.

The initialization order is defined only when there are no circular dependencies in the import graph.
Circular module dependencies cause all modules in the cycle to not have a defined initialization order relative to each other, though they are still guaranteed to be initialized before modules that depend on them but are not in the cycle.

Circular dependencies between initializer values are forbidden.

Example 1
import array; import io; void main() { for(season_num as i) { io.println(seasons[i]); } } array<str> get_seasons() { io.println("[seasons initialized]"); return { "Spring", "Summer", "Autumn", "Winter" }; } array<str> get_season_num() { io.println("[season_num initialized]"); return seasons->len(); } array<str> seasons = get_seasons(); uint season_num = get_season_num();
Output
[seasons initialized] [season_num initialized] Spring Summer Autumn Winter

import "test.lmn"; int e = set_and_log(12, "e"); int f = set_and_log(test.d + e, "f"); void main() { // Empty }
Output
initialized "d" initialized "e" initialized "f"

Global variables are initialized by a special __init function which cannot be called. This is where execution truly begins. After all global variables have been initialized, the main function is called.
