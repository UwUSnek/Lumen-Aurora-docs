Integral types are used to represent integers.
They are available in 4 sizes: byte (8 bits), int (32 bits), long (64 bits) and ext (128 bits). The programmer is responsible for choosing the size that best fits their use case.

Each size has a signed and an unsigned version.
Unsigned types cannot represent negative numbers, but have a larger maximum value than their signed counterpart.
Signed types use the Two's Complement to represent negative numbers.

Type Min Max
byte -128 127
ubyte 0 255
int -2,147,483,648 2,147,483,647
uint 0 4,294,967,295
long ~-9.223 × 10¹⁸ ~9.223 × 10¹⁸
ulong 0 ~1.845 × 10¹⁹
ext ~-1.701 × 10³⁸ ~1.701 × 10³⁸
uext 0 ~3.402 × 10³⁸

128-bit types (ext and uext) aren't natively supported in Lumen on most systems. Instead, they are simulated using multiple 64-bit values and operations. This makes them slower than their 64-bit counterparts.

128-bit types (ext and uext) are not supported by Aurora because most GPUs are simply not able to process them.
64-bit types (long and ulong) are often much slower than their 32-bit counterparts in Aurora because of missing hardware support. Smaller types should be preferred whenever possible.
