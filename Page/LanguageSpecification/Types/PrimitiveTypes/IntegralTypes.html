Integral types are used to represent integers.
Unsigned integral types cannot represent negative numbers, but the maximum value is twice as high as their signed counterpart.

All signed integral types use the Two's Complement to represent negative numbers.
Positive numbers and 0 use the corresponding binary value in both signed and unsigned integral types.

128-bit types (cent and ucent) aren't natively supported in Lumen on most systems. Instead, they are simulated using multiple 64-bit values and operations. This makes them slower than their 64-bit counterparts.

128-bit types (cent and ucent) are not supported by Aurora because most GPUs are simply not able to process them.
64-bit types (long and ulong) are often much slower than their 32-bit counterparts in Aurora because of missing hardware support. Smaller types should be preferred whenever possible.
