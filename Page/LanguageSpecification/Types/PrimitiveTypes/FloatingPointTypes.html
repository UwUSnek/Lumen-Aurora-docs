float, double, quad.
Floating point types are used to approximately represent real numbers.

All floating point types are encoded according to the IEEE 754 Standard.
Positive and negative inf, nan and signed 0 are all supported.
+0 and -0 are equal.

inf values are generated as required by the standard.
nan values are not generated for consistency and performance reasons.

The quad type isn't natively supported in Lumen on most systems. Instead, it's simulated using multiple double values and operations. This makes it slower than the double type.

The quad type is not supported by Aurora because most GPUs are simply not able to process it.
The double type is often much slower than float in Aurora because of missing hardware support. The float type should be preferred whenever possible.
