By default, all members of a struct are not visible from outside their module, meaning that routines cannot freely read or modify the value of individual members. This can be changed by using an Export directive inside of the struct's body.

Export directives allow a struct to define which members are accessible from the outside. Renaming members is allowed, though this only affects external references. The init, copy, and drop functions need to use the members' original name or aliases.

Routines declared in the same module as the struct type can freely access all of its members, either through their original name and aliases or using the ones created by the export directive.

Example - Cat.lmn
struct Cat { str name; str breed; str color; export name; } void toString(Cat cat) { // Same module, toString can access all members return cat.name + " is a " + cat.color + " " + cat.breed; }
Example - main.lmn
import io; include "Cat.lmn"; void main() { Cat c = { "Molecule", "British Shorthair", "grey" }; io.println(toString(c)); str name = c.name; // Ok, name is exported // str color = c.color; Error, color is not exported }
Output
Molecule is a grey British Shorthair
