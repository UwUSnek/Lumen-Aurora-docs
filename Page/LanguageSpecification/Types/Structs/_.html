Structs are types composed by multiple members that can have differing type.
They are especially useful for complex or non-uniform data that cannot be properly represented using primitive types.

Struct values can be copied and passed as a whole, just like values of primitive type, but they maintain their internal layout. Members can be accessed individually using the . keyword.

Example
import io; import list; struct User { str name; ulong id; bool admin; } List<User> users = { { "Snek", 0000, true } { "DangerNoodle", 0001, false } { "NopeRope", 0002, false } }; void main() { for(u in users) io.println( u.name + "\n\tID: " + u.id + "\n\tAdmin: " + u.admin + "\n\n" ); User firstUser = users[0]; io.println("The first user is: " + firstUser.name); }
Output
Snek ID: 0000 Admin: true DangerNoodle ID: 0001 Admin: false NopeRope ID: 0002 Admin: false The first user is: Snek

The size of a struct type is typically greater than the sum of sizes of the individual members. This is because of alignment requirements adding unused padding space between members. This doesn't affect the language in any way, but it might change how the data is represented in binary formats.

The compiler is allowed to freely reorder struct members or pack them by removing padding space in order to maximize runtime computational speed.
The compiler annotations :nopacking and :noreorder can be used to avoid this.
