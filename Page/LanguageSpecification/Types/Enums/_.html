Enums are types whose allowed values are a named subset of the possible values of their base type. The allowed values are called Enum elements.
Unlike most languages, the base type can be any type, including pointers, structs or other enums.
void cannot be used as base type.

Enum elements are implicitly const. They act like global variables and are part of the global initialization order.

Expressions of enum type are implicitly convertible to any type their base type is implicitly convertible to.
This is explained in detail in Implicit conversions.

Example
import io; enum Seasons : int { Spring; Summer; Autumn; Winter; } void main() { io.println("The " + Seasons:len + " seasons are:"); for(s in Seasons) { io.println("#" + (s:index + 1) + " " + s:name); } }
Output
The 4 seasons are: #1 Spring #2 Summer #3 Autumn #4 Winter

Internally, struct-based enum elements are no different from global const variables, while their values are equivalent to read-only pointers to the data and with special syntax.
This means that the size of a struct-based enum type is always 8 bytes, regardless of the size of the base type. Furthermore, creating or destroying enum values doesn't call any constructor or destructor of the base type.

Primitive-based enum types are the same size as their base type. Instead of pointers to global variables, they use the base value directly.
This doesn't affect global initialization order or other semantic rules in any way.
