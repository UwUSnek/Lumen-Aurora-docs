Enums with a struct as base type behave slightly differently than standard enums.

Elements are initialized through one of the constructors of the base type. If a destructor is defined, it is called before the process ends. This follows the global initialization order.

Example 1
import io; struct ResolutionData { uint width, height; uint pixels; str ratio; constructor(uint w, uint h, str r) { width = w; height = h; pixels = w * h; ratio = r; } } str toString(ResolutionData r) { return "" + r.width + "x" + r.height + ", " + r.pixels + " pixels, " + r.ratio; } enum Resolution : ResolutionData { HD = { 1280, 720, "16:9" } FullHD = { 1920, 1080, "16:9" } QuadHD = { 2560, 1440, "16:9" } UltraHD = { 3840, 2160, "16:9" } } void main() { io.println("FullHD: " + Resolution.FullHD->toString()); }
Output
FullHD: 3840x2160, 8294400 pixels, 16:9
Example 2
struct Color { ubyte r, g, b; } enum Palette : Color { Red = { 255, 0, 0 }; Green = { 0, 255, 0 }; Blue = { 0, 0, 255 }; White = { 255, 255, 255 }; Black = { 0, 0, 0 }; } void main() { for(c in Palette:elements) { io.println( "Color " + c:name + ": " + c.r + ", " + c.g + ", " + c.b ); } }
Output
Color Red: 255, 0, 0 Color Green: 0, 255, 0 Color Blue: 0, 0, 255 Color White: 255, 255, 255 Color Black: 0, 0, 0
