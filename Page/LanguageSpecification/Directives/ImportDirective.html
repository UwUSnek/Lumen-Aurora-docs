The Import directive allows the current module to access and rename symbols declared in other modules.
Only symbols exported using the Export directive and their children can be imported.

When an Import directive is used, it creates a new namespace in the root scope containing the imported symbols.
Anything these symbols depend on is not accessible unless explicitly imported.

The new namespace is named after the name of its file without extensions, or, if specified, the identifier used in the import directive.
An explicit identifier is required when the file name is not a valid identifier.

Import syntax Module interface name Module interface element Example 1
import "./test.lmn"; import io; void main(){ io.print(test.message); }
test.lmn
str message = "Hello :3";
Example 2
from math import sum; from io as console import print as write; void main(){ console.write(math.sum(8, 9, 10)); }
Equivalent code
// From the root scope's perspective, // this is what the program looks like // Root scope namespace { namespace math { // sum function... } namespace console { // print function ... [renamed to "write"] } // Module scope namespace { void main(){ console.write(math.sum(8, 9, 10)); } } }

Modules are always preprocessed and partially compiled before being imported.

Circular dependencies between modules are allowed, but global variables cannot create dependency loops. This is explained in more detail in Global initialization order
