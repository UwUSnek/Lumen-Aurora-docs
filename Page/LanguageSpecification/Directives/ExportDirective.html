The Export directive allows a module to define which symbols other modules are allowed to use when importing from it.

The Export directive can also be used to define which members of a struct type can be accessed from outside the struct. This is explained in detail in Structs.

Exporting a namespace also exports any exportable symbol it contains.
Any additional symbol an exported symbol depends on is automatically exported but hidden from the other modules.

Not specifying any symbol will export all the symbols in the module.
This doesn't allow renaming.

Export syntax Module interface element Example 1
float pi = 3.14; // Automatically exported as it is required by the "getTau" function, but not accessible float getTau(){ return pi*2; } export getTau as tau;
Example 2
struct vector3 { float x; float y; float z; export x, y, z; } export vector3;

The Export directive doesn't preserve the parent namespaces of a symbol nor can it add new ones.
This is usually not an issue, but it's possible to use Namespaces in conjunction with Symbol aliases to create a clean interface while preserving the internal structure of the module.

Example 3 - utils.lmn
from <math> import pow; // Use any namespaces you want to write and organize the module float pi = 3.14 float cube_v(float side) { return math.pow(side, 3) } float cube_s(float side) { return math.pow(side, 2) * 6; } float sphere_v(float r) { return pi / 3 * 4 * math.pow(r, 3); } float sphere_s(float r) { return pi * 4 * math.pow(r, 2); } // Create new namespaces for the export directive namespace volume { alias cube_v as cube; alias sphere_v as sphere; } namespace surface { alias cube_s as cube; alias sphere_s as sphere; } export volume, surface;
Example 3 - main.lmn
import <io>; import "utils"; void main(){ io.print(utils.volume.cube(3)); // Prints 27 }
