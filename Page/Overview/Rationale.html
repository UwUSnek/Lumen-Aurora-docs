Modern languages often trade runtime performance for simplicity, relying on the false assumption that whatever machine will be running the code is powerful enough to render the overhead negligible.
This creates a system where inefficiencies keep piling up, resulting in sub-optimal performance, even in the simplest of programs.

Furthermore, the combination of runtime data and complex semantics makes the debugging process needlessly time consuming.
This is often accompanied by an unreadable syntax and less-than-optimal error detection and reporting systems, all of which exacerbate the problem.

Lumen and Aurora exist to address these issues.
A shared static type system confines type-related issues to the compilation phase.
Simple semantics and a modular code structure simplify the debugging process and make it easier to keep a mental model of the program.
Templates and Directives allow for expressive but elegant syntax, and heavy optimization ensures maximum runtime performance.
