Comments are portions of code that are intentionally ignored.
They can be used to provide informations or temporarily exclude lines of code.
Comments are allowed to contain any sequence of Unicode characters of any length.

There are two types of comments: single line and multiline.
Single line comments start with // and end at the character that comes before the next character (\n).
Multiline comments start with /* and end with */.
Not closing a comment is allowed. This will cause it to include everything from its starting character sequence to the end of the file.

Comments cannot be nested.
Because they are removed before preprocessor directives are parsed, they also cannot be included through include statements or processed using preprocessor macros.

// Single line comment /* Multi / line / comment */

Because the character at the end of single-line comments is not considered part of the comment, it doesn't get removed by the preprocessor.
This means that single-line comments always separate tokens, making them unable to be used within macro definitions.
