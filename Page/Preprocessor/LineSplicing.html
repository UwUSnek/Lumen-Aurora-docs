NOTICE:
Splitting lines can easily lead to unreadable code. Proper language syntax should be preferred whenever possible.
Line splicing tokens are provided purely for completeness and familiarity. Use them responsibly.

Line splicing can be used to merge multiple lines and allow comments, include statements and language tokens to be written across multiple lines.

    The preprocessor supports two types of line splicing tokens:
    • Line continuation token: It consists of a \ immediately followed by a character.
      This type of line splicing token doesn't trim any whitespace. All adjacent tabs and spaces are preserved. Only the token itself (which includes the character) is removed.
    • Line junction token: It consists of two \ characters immediately followed by a character.
      This type of line splicing token trims any whitespace before it and at the start of the next line. This allows it to remove code indentation.

Line splicing tokens cannot be nested.
Using a line splicing token at the end of a file without a trailing is allowed and has no effect.

Line splicing tokens Example 1
// single line comment \ on two lines #include\ <ma\\ th> void fun\ ction() { uint va\\ r = 0, b = 0; v \\ ar + \\ \\ = b; }
Code after line splicing
// single line comment on two lines #include <math> void function() { uint var = 0, b = 0; var += b; }

Despite being removed after the include phase, line splicing tokens can be used anywhere in the source code, including in comments, include statements, and text literals. This is because all preprocessor parsers specifically look for and skip line splicing tokens.
