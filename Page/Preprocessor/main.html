The preprocessing phase precedes the code compilation.
It parses out line continuation tokens and comments and processes preprocessor directives.

Preprocessor directives are special instructions that allow the code of a module to be programmatically modified.
All the preprocessor directives are identified by the # character followed by their name and definition.
Each directive spans over a single line of code.
Preprocessor directives are always case sensitive.

Preprocessor operations are performed in a specific order:
  - Comments and line continuation tokens are removed.
  - macro definitions are evaluated and saved for later.
  - include statements are processed, merging all the files into one.
  - macro calls are replaced with the code they generated.
