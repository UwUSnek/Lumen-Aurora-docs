NOTICE:
The languages provide include statements, but they should be avoided whenever possible.
The import directive does a much better job at importing other modules, looks cleaner and can be debugged easily.
Include statements should only be used if you know what you are doing and you are certain that import directives won't be enough.

The include statement is used to paste code from other files.
Any existing file with read permissions can be included, regardless of its contents or extension.

An include statement consists of the sequence of characters #include followed by a string literal containing the path to the target file.
Any amount of whitespace may be present between the #include and the file path.
Standard modules can be included. They are accessed via the reserved ./standard path.
Because the path is a standard string literal, all escape sequences are available within it.

Absolute and relative paths are both supported. Relative paths are relative to the file the include statement is used in or any one of the specified include paths.
Glob patterns are also supported.

Syntax Example 1 - Single file includes
#include "/home/Snek/Lumen/file.lmn" // Absolute path #include "./file.lmn" // Relative path #include ":,@ why is this a file name" // Relative path #include "standard/math" // Includes the standard module "math" #include "./nonexistent_file" Error, non-glob patterns must correspond to an existing file #include "" Error, file name not specified
Example 2 - Glob patterns
#include "*.lmn" // Includes any file with lmn extension in the current directory #include "/home/Snek/Documents/**.lmn" // Includes any file with lmn extension in the Documents directory, recursively #include "generated/*" // Includes any file in the directory ./generated, but not the ones in sub-directories #include "generated/**" // Includes any file in the directory ./generated, recursively #include "?????" // Includes any file in the current directory whose name is 5 characters long (including the extension and the . character) #include "**" // Tries to include every file on your computer (the compiler will stop you)
Example 3
int main(){ #include"./generated/mainBody.txt" } // Writing code on the same line, either before the # character or after the path, is allowed.

Include operations are performed by replacing the include statement (from the # character to the string's closing ") with the contents of the included file.
Every byte of the source file is copied without modification, making include statements recursive.

    File paths have some limitations:
    • Files cannot be named . or ... . identifies the current directory, while .. identifies the parent directory.
    • Local directories called standard are not accessible. standard is a reserved name for the directory containing all standard modules.
    • File names cannot contain any of the following characters: *?\. These are reserved for glob pattern wildcards. Backslashes are not valid on Windows systems. Any other character is allowed.
    • ~ is not recognized as the home directory but as the literal character ~.
    • / identifies the root directory.
    When a glob pattern is used, all the source files are copied in the same way, one after the other, without adding separators in between.
    Matched files are copied in alphabetical order based on their absolute path.
    Glob patterns that don't match any file are allowed and have no effect.
    The preprocessor supports these glob pattern wildcards:
    •   ** - Any sequence of characters including / (any length, including 0).
    •   * - Any sequence of characters but / (any length, including 0).
    •   ? - Any character but /.

The last variable-length wildcard (* or **) of a glob pattern stops before the last occurrence of the character or wildcard that follows them.
Variable-length wildcards that are not last, instead, stop before the very first occurrence of the character or wildcard that follows them.
They are evaluated from left to right.

Glob patterns never match standard modules.

Symlinks in the included path are always resolved.

#include "./*.*" /* This Glob pattern will match all of these: │ │ ./file.txt ./.txt ./file. ./file.txt.tmp.zip 1 2 ╰┬─╯ ╰┬╯ ╰┬╯ ╰┬─╯ ╰┬─╯ ╰────┬────╯ 1 2 2 1 1 2 */
