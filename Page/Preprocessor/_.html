Traditionally, the preprocessor is a separate program invoked before compilation. It is responsible for splicing lines, removing comments, and executing preprocessor directives.
The ALC compiler implements the preprocessor as a compilation phase. This lets it generate better diagnostics and reduce overhead.

This phase is fully independent and works regardless of the languages' lexicon and semantics.
At this stage, tokens haven't been formed yet. The entire source code is treated as a stream of individual characters.

    Preprocessor operations are performed in the following order:
    • 1. Include phase.
      Include statements are recursively replaced with the contents of the specified file.
    • 2. Line splicing.
      Line splicing tokens are removed, merging affected lines.
    • 3. Cleanup phase.
      All comments are removed.
