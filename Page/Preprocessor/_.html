The preprocessing phase precedes the compilation of the source code.
It is responsible for splicing lines, removing comments and executing preprocessor directives.

Preprocessor directives are special instructions that allow the code of a module to be programmatically modified.
All preprocessor directives are identified by the # character followed by their name and definition/parameters.
The name of preprocessor directives is always case sensitive.

This phase is fully independent and works regardless of the code's correctness.
At this stage, the concept of token doesn't exist yet. The entire source code is treated as a stream of single characters. This allows the preprocessor to freely edit whitespace and create or split tokens.

    Preprocessor operations are performed in the following order:
    • Include phase
    • Line splicing
    • Cleanup phase
    • Macro substitution
